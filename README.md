本项目是针对“闪动校园” APP 开发的自动化工具集。包含全自动登录（自动破解滑动验证码）、个人数据面板查询、历史记录抓取，以及基于地理围栏的高仿真模拟跑步（Ghost Running）功能。
✨ 核心特性

    🔐 全自动免密登录：内置 AES 加密解密算法，结合 OpenCV 图像边缘检测（Canny）与模板匹配，全自动破解滑动拼图验证码。
    👥 多账号无缝切换：登录成功后自动将 Token 等鉴权信息本地持久化到 config.json，后续操作（多账号）可直接读取缓存，无需重复登录。
    📊 全息数据面板：一键拉取学校跑步考核规则（周期、配速限制、单次里程限制等）、个人当前完成进度以及历史打卡记录。
    📍 智能地理围栏解析：自动获取当前学校的电子围栏边界多边形（Polygon），作为模拟跑步的地理约束。
    🏃 高仿真幽灵跑（Ghost Runner）：
        动态路线规划：算法自动读取当前任务的“必经打卡点”，并以围栏中心为终点规划路线，保证 100% 触发打卡桩。
        真实物理模拟：模拟真实的 GPS 坐标漂移、步数推算（结合步频）、随机卫星数量、配速控制（默认 ~5分钟/公里）。
        心跳与风控绕过：严格遵守官方的心跳包发送频率（每 5 个坐标点上传一次），并自动继承原始人脸底图数据绕过活体检测。

🛠️ 环境依赖

确保你的系统已安装 Python 3.8+，然后安装以下依赖库：

pip install curl_cffi pycryptodome opencv-python numpy

    说明：本项目使用 curl_cffi 模拟真实的 Chrome TLS 指纹，以绕过常规的爬虫检测。

🚀 快速开始

本项目包含两个主要脚本模块：
1. 账号初始化与数据查询 (shanding_app.py)

该脚本用于首次登录、生成本地配置缓存，以及查询你的跑步进度。

    打开脚本，定位到末尾的 if __name__ == "__main__":。
    填入你的手机号和密码。
    运行脚本：

    python shanding_app.py

    脚本会自动完成滑动验证码破解、登录鉴权，并在同目录下生成 config.json，同时打印出你的学期规则和历史记录。

2. 自动化模拟跑步 (ghost_runner.py)

依赖 config.json，真正执行自动化“代跑”的脚本。

    确保同目录下已存在 config.json。
    运行脚本：

    python ghost_runner.py

    根据终端提示，输入数字选择要跑的账号。
    脚本会自动拉取学校的电子围栏列表，选择你要跑步的校区/区域。
    脚本将开始工作，默认以真实的物理时间速率执行 15 分钟（约 3 公里） 的模拟跑步。
    期间请勿关闭程序，完成后会自动进行结算、上传轨迹点与步数。

🧩 核心技术架构简述

    签名机制校验：请求体中除 sign 外的所有参数按字典序排序拼接，通过 SHA256 哈希后，再经由 AES-CBC (Key: RHXL...) 加密生成数字签名。
    滑动验证码轨迹：提取真实偏移量 X 后，通过线性变换 int(X - 6.5) + 16.5 构造伪造坐标，并使用 AES-ECB 进行加密提交。
    坐标系移动算法：根据经纬度使用 Haversine 公式计算实际物理距离（米），通过向量比例平滑推算下一步的伪造 GPS 坐标，模拟匀速运动。
